    Balance transfer

    余额代偿

    我没有答应申请, 因为这个余额代偿我暂时不需要. 余额代偿就简单来说就是把一张信用卡欠的钱转移到另一张卡上. 或者用一张信用卡去偿还另一张信用卡. 为啥要搞这么麻烦呢? 比如 你有两张信用卡A和B, A这个月需要偿还1000英镑, 可是你没有这么多钱, 根据信用卡规定 你即使每个月无法所有还清 也许要还一小部分钱 (一般是按5% 最低5英镑) 也就是说 最小偿还(Minimal Payment)是50英镑. 那么剩下950英镑你可能需要在接下来的几个月还清, 但是银行会收你利息 (一般来说54天内还清是不收利息的). 碰巧另一张卡B有个1年内免息的活动. 所以你这时候就可以把A卡所欠的债给转移到B卡上, 这样在1年后还清B卡的债就可以了. 这也就相当于免息向银行借钱. 一般来说余额代偿是为了更好的利用银行信用卡的活动来达到少还利息的目的(低的利息甚至是零利息).

    当然, 如果每个月都还清的 (Full Payment) 就用不上这个福利了. 银行也不可能一直都是 0%的 余额代偿, 在搞活动之后就会收取手续费. 所以你想用一个坑填补另一个洞的想法是不太现实的, 至少这种方法持久是不实际的. 短期内也许可以暂时帮你解决燃眉之急 但是出来混迟早是要还的, 欠的钱总有一天要还清, 后面利息越来越高(滚雪球), 这个大洞迟早要大漏的.

    而且如果太多信用卡 并且有经常搞余额代偿就很可能会降低你的信用分数. 我用信用卡最看重的是积分积点换优惠, 所以我每次都拒绝HSBC的好意, 都会和他们说我已经有AMEX美国运通卡.

    A sample Node.js app to demonstrate fabric-client & fabric-ca-client Node.js SDK APIs

    一个示例Node.js应用程序，用于演示fabric-client 和 fabric-ca-client Node.js SDK API

    Prerequisites and setup:

    要求和设置 :

    • Docker - v1.12 or higher
    • Docker Compose - v1.8 or higher
    • Git client - needed for clone commands
    • Node.js v8.4.0 or higher
    • Download Docker images

    Once you have completed the above setup, you will have provisioned a local network with the following docker container configuration:

    完成上述设置后，您将使用以下Docker容器配置配置本地网络：

    • 2 CAs
    • A SOLO orderer
    • 4 peers (2 peers per Org)

    Artifacts

    • Crypto material has been generated using the cryptogen tool from Hyperledger Fabric and mounted to all peers, the orderering node and CA containers. More details regarding the cryptogen tool are available here.

    加密资料是使用Hyperledger Fabric的cryptogen 工具生成的，并已安装到所有对等方，排序节点和CA容器中。 可在[此处]（http://hyperledger-fabric.readthedocs.io/en/latest/build_network.html#crypto-generator）获得有关cryptogen工具的更多详细信息。

    • An Orderer genesis block (genesis.block) and channel configuration transaction (mychannel.tx) has been pre generated using the configtxgen tool from Hyperledger Fabric and placed within the artifacts folder. More details regarding the configtxgen tool are available here.

    orderer创世区块（genesis.block）和通道（mychannel.tx）已被预使用从Hyperledger fabric的 configtxgen工具产生并且放置的工件夹内。可用[这里]关于configtxgen工具更多细节（http://hyperledger-fabric.readthedocs.io/en/latest/build_network.html#configuration-transaction-generator）。

    Network configuration considerations

    网络配置注意事项

    You have the ability to change configuration parameters by either directly editing the network-config.yaml file or provide an additional file for an alternative target network. The app uses an optional environment variable “TARGET_NETWORK” to control the configuration files to use. For example, if you deployed the target network on Amazon Web Services EC2, you can add a file “network-config-aws.yaml”, and set the “TARGET_NETWORK” environment to ‘aws’. The app will pick up the settings inside the “network-config-aws.yaml” file.

    您可以通过直接编辑network-config.yaml文件或为备用目标网络提供其他文件来更改配置参数。 该应用程序使用可选的环境变量“ TARGET_NETWORK”来控制要使用的配置文件。 例如，如果您在Amazon Web Services EC2上部署了目标网络，则可以添加文件“ network-config-aws.yaml”，并将“ TARGET_NETWORK”环境设置为“ aws”。 该应用程序将获取“ network-config-aws.yaml”文件中的设置。
